What is the difference between XSLT 1.0 and XSLT 2.0?

Simplified stylesheets

2.0 allows for simplified stylesheets for cases where stylesheets only have one template rule for the root node: one can combine content from xsl:stylesheet and xsl:template elements into the opening html tag. This eliminates the xsl: stylesheet/template/output elements.

Multiple output documents

2.0 allows for multiple output documents per processor run.

User defined functions

2.0 allows for creating and calling user defined functions.

Grouping

2.0 has improved grouping functionality supported by built-in functions.

Validating output

2.0 has the ability to validate the output of an XSLT transformation against an XML schema.

What is the difference between XPath 1.0 and XPath 2.0?

Data model

1.0 works on node sets. 2.0 is based on sequences: nodes and/or values. Sequences also allow for duplicate values.

Strongly-typed

2.0 is strongly typed: "use of values is strictly dependent on each values type" and supports XML schema. Additional functions avg(), min() and max().

Additional functions

avg(), min(), max(), upper-case(), lower-case(), exists(), distinct-values(), current-date(), current-time, current-dateTime(), unparsed-text(href), tokenize(input, regex_pattern), replace(input, regex_pattern, replace), match(input, regex_pattern), analyze-string()

Testing, Quantifying and Looping

2.0 allows for testing of conditions within the XPath expression; evaluating for boolean and use of if-then-else as opposed to xsl:choose. Also allows for looping over sequences: similar to xsl:for-each.
